# blog-outline

ブログ執筆ワークフローの 第2段階。
`ideas/{slug}.md` を入力に、記事の骨組み（タイトル・リード方向・見出し構造）を作る。

## 入力

- 引数: `{slug}`（省略時は `ideas/` を一覧表示してユーザーに選ばせる）
- 読み込み: `ideas/{slug}.md`

## 進め方

1. コンセプトの再確認
   - `ideas/{slug}.md` を読み、一文コンセプト・読者像・独自角度を要約してユーザーに提示
   - 「この理解で合っていますか？」と確認。ズレていれば ideate に戻ることを提案

2. タイトル案 3 つ
   - 異なる角度の 3 案を提示（例: 体験談寄り／ハウツー寄り／問題提起寄り）
   - 釣りタイトルや AI 臭の強い言い回しは避ける
   - ユーザーに選ばせる、もしくはハイブリッド案を一緒に練る

3. リード文の方向性
   - 本文は書かない。「最初の3〜4文で何を伝えるか」を箇条書きで合意
   - 読者が「これは自分の話だ」と思える掴みを優先

4. H2/H3 構造
   - 3〜5 個の H2 を提案。各 H2 に「要点」「含める要素」「使う一次体験」のメモを付ける
   - 1記事に詰め込みすぎない。書ききれない論点は「ボツ案・脇道メモ」へ退避
   - 順序は読者の問いの自然な流れになるよう確認

5. 保存
   - `.claude/skills/blog-outline/templates/outline.md` をベースに埋める
   - 出力先: `drafts/{slug}/outline.md`（ディレクトリも `{slug}` のまま、日付なし）
   - frontmatter の `status: outline` に更新

## 重要原則

- 構造の押し付けをしない。ユーザーが書きたい順序を尊重
- 「網羅性」より「一本の通った道筋」を優先
- ボツ案を消さない。後で別記事の種になる
- セクション数で迷ったら少ない方を選ぶ

## 次のステップ

完了後、ユーザーに `/blog-draft {slug}` で本文執筆段階に進めることを伝える。
