# blog-ideate

このリポジトリのブログ執筆ワークフローの 第1段階。
コンセプトをゼロから対話で発掘し、`ideas/{slug}.md` に保存するスキル。

## 前提

- 対象は「文章が苦手だがAI時代の開発について書きたい」ユーザー
- 一次体験（自分が手を動かして驚いた／詰まった／工夫したこと）を引き出すのが目的
- 抽象論やトレンド解説の量産はしない

## 進め方

1. 挨拶と現在地確認（1往復）
   - 今日「書きたい気分の種」が既にあるか、ゼロからかを聞く
   - ある場合は早めに核心へ。ない場合は質問駆動で掘る

2. ヒアリング（5〜7問、1問ずつ）
   一気に全部聞かず、回答を見て次を出す。例:
   - 「最近 1〜2 週間で AI／開発まわりで一番手が止まった瞬間は？」
   - 「逆に予想を超えてうまくいったことは？」
   - 「同僚や友人に思わず話したくなったエピソードは？」
   - 「『これ誰か書いてくれてないかな』と検索して見つからなかったテーマは？」
   - 「自分なら他の人と違う角度で語れそうな経験は？」

3. 読者像の絞り込み
   - 「この話は誰に一番刺さるか？」を 1〜2 行で言語化させる
   - 「すでに知っている人」と「これから触る人」のどちらに寄せるか確定

4. 独自角度の磨き込み
   - 「同じテーマで既に存在する記事と何が違う？」を必ず質問する
   - 一次体験・具体的な数字・コードでの観測など、再現できない要素を引き出す

5. 保存
   - スラグを `kebab-case-title` で提案 → ユーザー確認
   - ファイル名に日付は付けない（`ideas/{slug}.md`）
   - `.claude/skills/blog-ideate/templates/concept.md` をベースに埋めて書き出す
   - `created` には今日の日付（YYYY-MM-DD）

## 出力

- `ideas/{slug}.md`
- frontmatter の `status: idea` を維持
- 「ブレストメモ」欄には対話で出た断片を整形しすぎず残す（次段階の素材）

## 重要原則

- 書き手を代替しない。質問して引き出すのが役割
- 1メッセージで複数の質問を畳みかけない（圧迫感を出さない）
- ユーザーの言葉をそのまま使う。AI 風の言い換えを勝手にしない
- ネタが弱いと感じても無理に書かせず、「今日は寝かせて別のネタを探す」選択肢も提示する

## 次のステップ

完了後、ユーザーに `/blog-outline {slug}` で構成段階に進めることを伝える。
